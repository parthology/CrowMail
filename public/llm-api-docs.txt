# CrowMail API Reference
# Base URL: https://api.crowmail.sbs
# Authentication: Bearer Token (per-user JWT) or Admin API Key (dk_xxx)
# This file is designed to be sent to AI assistants for integration help.

---

## Authentication

### Bearer Token (per-user)
Obtain a token via POST /token with email address and password.
Include in requests: Authorization: Bearer <token>
Tokens are per-account and expire in 7 days.

### Admin API Key (server operator)
Static key configured in server .env (ADMIN_API_KEYS). Grants access to /admin/* endpoints.
Format: dk_ prefix. Include in requests: Authorization: Bearer dk_xxx

### Inbound Webhook Secret
Shared secret used by the /admin/inbound-webhook endpoint. Configured on both the CrowMail backend (INBOUND_WEBHOOK_SECRET env var) and the Cloudflare Email Worker (X-Webhook-Secret header). Not a user-facing credential.

## Endpoints

### [Domains] GET /domains
List domains that CrowMail is configured to receive mail for.
Auth: None
Query: page (default 1)
Response:
{
  "hydra:member": [
    {
      "id": "string",
      "domain": "crowmail.sbs",
      "isVerified": true,
      "ownerId": null,
      "createdAt": "2026-01-01T00:00:00Z",
      "updatedAt": "2026-01-01T00:00:00Z"
    }
  ],
  "hydra:totalItems": 1
}

### [Auth] POST /token
Get authentication token using email address and password.
Auth: None
Request:
{
  "address": "user@crowmail.sbs",
  "password": "your_password"
}
Response (200):
{ "id": "account-id", "token": "eyJ..." }
Errors:
- 401 Invalid email or password.

### [Accounts] POST /accounts
Create a new email account.
Auth: None
Request:
{
  "address": "user@crowmail.sbs",
  "password": "your_password",
  "expiresIn": 0
}
Fields:
- address (required): email under a domain in /domains. Username (before @) >= 3 chars.
- password (required): >= 6 chars.
- expiresIn (optional integer seconds): 0 or -1 = never expires; omit = 24h default; >0 = custom expiry.
Response (201):
{
  "id": "string",
  "address": "user@crowmail.sbs",
  "quota": 1073741824,
  "used": 0,
  "isDisabled": false,
  "isDeleted": false,
  "username": "user",
  "createdAt": "2026-01-01T00:00:00Z",
  "updatedAt": "2026-01-01T00:00:00Z"
}
Errors:
- 400 Domain not allowed / invalid address / password too short.
- 409 Address already in use.

### [Accounts] GET /me
Get the currently authenticated account.
Auth: Required (Bearer Token)
Response (200): same shape as POST /accounts response.

### [Accounts] DELETE /accounts/{id}
Delete your own account by ID. You may only delete the currently logged-in account.
Auth: Required (Bearer Token)
Response: 204 No Content

### [Accounts] PATCH /accounts/me/password
Change the password of the currently authenticated account.
Auth: Required (Bearer Token)
Request:
{
  "old_password": "current",
  "new_password": "new-password-at-least-8-chars"
}
Response (200): { "ok": true }
Errors:
- 400 New password too short (< 8 chars).
- 401 Invalid password (old_password wrong).
After a successful change all existing tokens keep working until they expire; recommended to POST /token again to refresh.

### [Messages] GET /messages
Get inbox message list (paginated, newest first). List view omits full body.
Auth: Required (Bearer Token)
Query: page (default 1)
Response:
{
  "hydra:member": [
    {
      "id": "string",
      "accountId": "string",
      "msgid": "string",
      "from": { "name": "Sender", "address": "sender@example.com" },
      "to": [ { "name": "You", "address": "you@crowmail.sbs" } ],
      "subject": "Email Subject",
      "intro": "First ~200 chars of body...",
      "seen": false,
      "isDeleted": false,
      "hasAttachments": false,
      "size": 1024,
      "downloadUrl": "https://api.crowmail.sbs/sources/<id>",
      "createdAt": "2026-01-01T00:00:00Z",
      "updatedAt": "2026-01-01T00:00:00Z"
    }
  ],
  "hydra:totalItems": 5
}

### [Messages] GET /messages/{id}
Get full message details including HTML/text body.
Auth: Required (Bearer Token)
Response (200):
{
  "id": "string",
  "accountId": "string",
  "from": { "name": "Sender", "address": "sender@example.com" },
  "to": [ { "name": "You", "address": "you@crowmail.sbs" } ],
  "subject": "Email Subject",
  "text": "Plain text body",
  "html": [ "<html>...</html>" ],
  "seen": false,
  "isDeleted": false,
  "hasAttachments": false,
  "size": 2048,
  "downloadUrl": "https://api.crowmail.sbs/sources/<id>",
  "createdAt": "...",
  "updatedAt": "..."
}
Errors: 404 if message not found or belongs to another account.

### [Messages] PATCH /messages/{id}
Mark a message as read.
Auth: Required (Bearer Token)
Response: { "seen": true }

### [Messages] DELETE /messages/{id}
Soft-delete a message (sets is_deleted=1; nightly cron hard-purges).
Auth: Required (Bearer Token)
Response: 204 No Content

### [Messages] GET /sources/{id}
Get raw RFC-822 message source.
Auth: Required (Bearer Token)
Response: message/rfc822 body.

## Admin / operator endpoints

### [Admin] POST /admin/inbound-webhook
Accept an already-received email from a trusted forwarder (Cloudflare Email Worker in the reference deployment). Bypasses SMTP.
Auth: X-Webhook-Secret header matching INBOUND_WEBHOOK_SECRET env var.
Request:
{
  "to": "recipient@crowmail.sbs",
  "from_address": "noreply@discord.com",
  "from_name": "Discord",
  "subject": "Verify your email",
  "html": "<html>...</html>",
  "text": "plain body",
  "raw": "<full RFC-822 source>"
}
Behavior:
- to must be under an allowed domain.
- Account is auto-provisioned if it does not yet exist (temp-mail semantics).
- If subject/html/text are empty, the backend parses `raw` to extract them (via mail-parser).
- SPF/DKIM/DMARC checks are skipped (Cloudflare validates before invoking the Worker).
Response (200): { "ok": true, "message_id": "<uuid>" }
Errors:
- 400 unknown domain / invalid recipient / mailbox full.
- 401 Missing or wrong X-Webhook-Secret.

## Real-time updates (SSE)

Server-sent events for new-message notifications:
GET /events?topic=/accounts/{id}
Auth: Bearer Token in Authorization header OR ?token=... query.
Emits JSON events on new mail; use as replacement for polling.

---

## Error Response Format

{
  "error": "Error Type",
  "message": "Detailed error message",
  "detail": "Same as message in most cases"
}

Status codes:
- 400 Bad Request (invalid body / domain not allowed / password too short)
- 401 Unauthorized (missing/wrong token or wrong password)
- 403 Forbidden (deleting another user's data)
- 404 Not Found
- 409 Conflict (address already in use)
- 429 Too Many Requests (rate limit)
- 500 Internal Server Error

Rate limit: RATE_LIMIT_QPS per source IP (default 100 in the reference deployment).

## Quick Start Example

# 1. Create account (expiresIn: omit=24h, 0/-1=never, >0=custom seconds)
curl -X POST https://api.crowmail.sbs/accounts \
  -H "Content-Type: application/json" \
  -d '{"address":"test@crowmail.sbs","password":"MyPassword1!","expiresIn":0}'

# 2. Get token
TOKEN=$(curl -s -X POST https://api.crowmail.sbs/token \
  -H "Content-Type: application/json" \
  -d '{"address":"test@crowmail.sbs","password":"MyPassword1!"}' | jq -r .token)

# 3. Read inbox
curl https://api.crowmail.sbs/messages -H "Authorization: Bearer $TOKEN"

# 4. Get one mail's HTML body
curl https://api.crowmail.sbs/messages/<id> -H "Authorization: Bearer $TOKEN"

# 5. Change password
curl -X PATCH https://api.crowmail.sbs/accounts/me/password \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer $TOKEN" \
  -d '{"old_password":"MyPassword1!","new_password":"NewPass1234!"}'

# 6. Delete account
ACCOUNT_ID=$(curl -s https://api.crowmail.sbs/me -H "Authorization: Bearer $TOKEN" | jq -r .id)
curl -X DELETE "https://api.crowmail.sbs/accounts/$ACCOUNT_ID" \
  -H "Authorization: Bearer $TOKEN"
